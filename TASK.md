# TASK.md

## Session checklist

1. Read `/specs/poker-trainer-system/proposal.md`, `requirements.md`, `design.md`, and `tasks.md`.
2. Confirm the current implementation phase before changing code or content.
3. Work only on the in-scope module or deliverable for that phase.
4. Preserve the planned folder structure and shared scenario vocabulary.
5. Check poker assumptions before implementing logic, especially when guidance is simplified.
6. Test manually for the changed flow and confirm empty/error states still make sense.
7. Update docs when product scope, scenario schema, or trainer behavior changes.
8. Use clean commit messages and avoid unrelated edits.

## Commit style

- `feat: add preflop trainer session shell`
- `fix: correct pot odds answer feedback`
- `docs: update postflop handbook examples`
- `refactor: split scenario explanation helpers`

## Priority order

1. Correct poker logic and clear assumptions
2. Stable training flow and understandable feedback
3. Maintainable structure and small modules
4. Visual polish and performance improvements
